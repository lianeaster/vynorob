# Винороб — Система створення технологічних схем виноробства

Веб-додаток для покрокового створення принципових технологічних схем виноробства з урахуванням кольору, стилю, CO₂, кислотності, сенсорного та технічного профілю.

## Можливості

- Покрокове створення технологічної схеми вина (7 кроків).
- Автоматичний підбір базового маршруту (біла, червона, рожева, orange, blanc de noirs).
- Модульна система: модулі CO₂, TA-корекції, дуба, MLF, стабілізації активуються за параметрами.
- Генерація абстрактного графа схеми (nodes + edges) та mermaid-діаграми.
- Рендер на canvas з експортом у PNG та PDF.
- Авторизація, збереження схем, відновлення незавершених сесій.

## Кроки створення схеми

| Крок | Назва | Що задається |
|------|-------|--------------|
| 1 | Колір / схема бродіння | Очікуваний стиль кольору, схема екстракції (біла, червона, рожева, orange, blanc de noirs) |
| 2 | Стиль | Категорія солодкості (сухе/напівсухе/напівсолодке/десертне), цільова спиртуозність, S₀, RS, спосіб отримання солодкості, стабільність при цукрі |
| 3 | Стиль за CO₂ | Тихе/перлант/фріззанте/ігристе, цільовий тиск, метод CO₂ (природний/карбонізація/танк/пляшка/пет-нат), тара, цукор на вторинне бродіння |
| 4 | Кондиції сировини (TA) | Поточна TA, цільова TA, референс-норма (ДСТУ 4806/4807), режим корекції, відповідність нормам |
| 5 | Бажаний смаковий стиль | Сприйняття кислотності, тіло, танінність, гіркота, ароматика (фрукти, ягоди, тропічні, квітки, пряності), дуб, окиснювальний/редуктивний стиль, MLF, кремовість |
| 6 | Технічні параметри | pH, YAN, SO₂-режим, температура бродіння, живлення дріжджів, контакт з осадом, батонаж, білкова/тартратна стабілізація, фільтрація, мікро- та VA-ризики |
| 7 | Підсумок | Перегляд усіх параметрів, створення технологічної схеми |

## Логіка побудови схеми

Від параметрів кроків 1–6 до готової схеми:

1. **Нормалізація атрибутів** — з `wine_data` обчислюються: `color`, `sweetness_category`, `alcohol_category`, `CO2_level`, `CO2_method`, `TA_profile`, `sensory_profile_tags`, `tech_profile_tags`.
2. **Підбір базової схеми** — за кольором, стилем, CO₂ та схемою бродіння обирається базовий маршрут (`white_classic`, `red_classic`, `rose_skin_contact`, `orange_skin_contact` тощо).
3. **Активація модулів** — за умовами TA, CO₂, сенсорики та техніки вмикаються додаткові блоки (корекція TA, вторинне бродіння, дуб, MLF, стабілізація).
4. **Побудова графа** — формується абстрактний граф `{nodes, edges}`.
5. **Підстановка параметрів** — у підписи блоків підставляються числові значення (t, TA, SO₂, S₀, RS).
6. **Рендер** — граф рендериться на canvas та/або як mermaid-діаграма.

Діаграма логіки доступна у файлі `scheme_logic.html`.

## Технології

- **Backend:** Python 3 + Flask
- **Frontend:** HTML, CSS, JavaScript (vanilla)
- **Canvas рендер:** `SchemeRenderer` (custom canvas drawing)
- **База даних:** JSON файли (`data/`)
- **Авторизація:** модуль `auth.py` (сесії Flask)

## Встановлення

```bash
pip install -r requirements.txt
python3 app.py
```

Відкрийте браузер: http://localhost:5000

|-------|-----|------|
| GET | `/` | Головна (редірект) |

| GET | `/style` | Крок 2: Стиль |
| GET | `/style-co2` | Крок 3: CO₂ |

| GET | `/style-tech` | Крок 6: Техніка |
